What It Does

8 Core Capabilities

Tap any card to see technical details and real-world business applications.

💓
The Heartbeat Engine
Self-renewing 9-phase cycle, every 2 hours
⌄
A 9-phase cycle runs every 2 hours via CronCreate. Phase A detects file changes (cheap). Phases B and C gather live data from MCP servers. Phase D runs cross-domain analysis. Phase E rebuilds the dashboard. Phase G self-renews the cron job before it expires. The system maintains itself indefinitely without manual intervention.
🏢 Business Application
Operations centers, security monitoring, compliance tracking, automated reporting cadences that never need a maintenance window.
🧠
12 Specialized Skills
Markdown contracts, organic creation
⌄
Each skill is a markdown file: input contract, output contract, failure conditions. Skills are invoked by the Claude runtime — not a scheduler. A skill is only created after the same task has been done manually 3 times. No premature abstraction. Current skills cover: knowledge capture, email drafting, meeting prep, iterative research, content generation, and career tracking.
🏢 Business Application
Institutional knowledge capture, onboarding acceleration, reusable process playbooks that any future AI runtime can execute without re-engineering.
⏰
5 Daily Automated Tasks
Time-gated, state-machine controlled
⌄
Each task checks three conditions before running: (1) already ran today? (2) is it the right hour? (3) is it the right day of week? Tasks: morning digest at 06:00 daily, career pulse at 08:00 weekdays, opportunity scanner at 10:00 weekdays, network pulse at 14:00 Mon/Wed/Fri, calendar intelligence at 19:00 daily. A state file prevents duplicate runs.
🏢 Business Application
Tender tracking, market intelligence, compliance deadlines, automated briefing preparation — all without a dedicated scheduler service or cloud function.
📚
Knowledge Management
Obsidian-lite: markdown and quality gates
⌄
Three knowledge layers: eureka captures (strategic insights, tagged), digests (daily news filtered by 3 quality gates: day-precise date, clickable URL, non-obvious finding), and research briefs (iterative scored research with coverage/relevance/actionability rubric, max 3 rounds). No database or vector search required.
🏢 Business Application
Due diligence, competitive intelligence, market analysis. Quality gates ensure no hallucinated sources or stale articles reach the output layer.
🗂️
Context File System
Markdown prompts that replace fine-tuning
⌄
A set of markdown files injected at session start that define who, what, where, and why for the AI runtime. Covers identity, current work, team, priorities, goals, and voice profile. Updated by the brain-drainer skill at session end. Replaces expensive fine-tuning with cheap prompt injection. Any change immediately affects all future sessions.
🏢 Business Application
Team onboarding, institutional memory, collective knowledge base, handoff documentation — readable by humans and executable by AI alike.
🔄
Self-Improvement Engine
Corrections learned and persisted across sessions
⌄
A learned-rules file captures behavioral corrections in real time: format, tone, workflow, content standards. Continuous improvement rules define weekly audit cycles and monthly system reviews. A decision log is append-only, providing a complete audit trail. Corrections made once apply permanently across all future sessions without retraining.
🏢 Business Application
Continuous process optimization, quality management, audit trails, kaizen cycles — the system genuinely improves from operational feedback.
🔌
MCP Integration Hub
LLM as the integration layer
⌄
Over 10 MCP servers connected: email, calendar, task management, code repositories, messaging, job search, browser automation. Tool access is scoped per-agent via a config file. New integrations added without code changes — add a server definition and the runtime discovers the tools. The LLM handles context, routing, and error recovery.
🏢 Business Application
Enterprise integration without middleware. The LLM replaces Zapier, n8n, and custom connectors for workflows that require reasoning, not just data routing.
📊
Command Center Dashboard
Single-file HTML, rebuilt each active cycle
⌄
A single HTML file updated by the heartbeat engine on every active cycle. A DATA object holds all dynamic content: metrics, contact scores, alerts, heartbeat log entries. Readable in any browser. Sharable as a file attachment. No server, no build tools, no hosting required. The HTML file is the database, frontend, and report simultaneously.
🏢 Business Application
Portable operational dashboards, zero-maintenance reporting, stakeholder briefing tools — deployable with zero infrastructure and zero ongoing cost.

Design Thinking

7 Architecture Principles

The constraints that make this system maintainable, cheap, and genuinely useful.

01
Claude Code-native platform
The LLM runtime IS the platform. No separate orchestrator, no agent framework, no pipeline DSL. Skills are markdown files. State is JSON. The REPL is the scheduler.
02
Pure markdown + JSON + HTML
Fully portable. Zero dependencies on hosted services beyond the AI API itself. Move it to a new machine: copy one folder and it works. No lock-in.
03
Organic skill creation: 3x-then-build
No premature abstraction. A skill is only created after the same task has been done manually 3 times. Every skill is born from real usage, not speculation about future needs.
04
State machine patterns for reliability
Formal states: active / paused / pending / completed. Error counts. Last-run timestamps. Prevents duplicate runs, detects failures, enables recovery without human intervention.
05
Self-renewal by design
The system recreates its own scheduled jobs before they expire. Zero manual maintenance after initial setup. The heartbeat runs until explicitly stopped.
06
Cost transparency per cycle
$0.17 per heartbeat cycle. ~$14/month total across all automations. Every task has a measurable cost per run. AI automation is cheap — but measure it anyway.
07
Quality gates at every output stage
Day-precise publication dates on news. Scoring rubric on research (coverage/relevance/actionability). Fact-check on public-facing content before it ships. No hallucinations in the output layer.
Built with Claude Code. No Docker, no SQL, no build tools.

"/loop" / "start the loop"
Continuous intelligence heartbeat — the core automation engine. Self-renewing, session-scoped, absorbs multiple other automation functions.
Input
All context and state files
Output
Updated command-center.html + console output
Detail
9-phase cycle, 2-hour interval via CronCreate. Registers its own cron job and renews it at 60h (before the 72h expiry). Absorbs: calendar prep, health nudges, weekly review elements. Cost: ~$0.17/cycle.
Trigger
"research X" / "deep dive on"
Iterative scored web research with quality threshold. Stops when findings are good enough. Flags gaps explicitly when not.
Input
Research topic + current priorities context
Output
knowledge/research/YYYY-MM-DD_slug.md
Detail
Scoring rubric: coverage (3pts) + relevance (4pts) + actionability (3pts). Score under 6: iterate with refined queries. Max 3 rounds. INSIST items surface immediately when a finding requires same-day action.
Trigger
"think ahead" / "what am I missing"
Anticipatory analysis across 5 domains. Surfaces non-obvious insights with specific actions — not vague observations.

All context + latest automation logs
Output
knowledge/thinking/YYYY-MM-DD.md + top 3 insights
Detail
5 domains: career trajectory, content opportunities, relationship intelligence, goal progress, system optimization. Every insight must include a specific action. Runs embedded in heartbeat Phase D or on-demand.
